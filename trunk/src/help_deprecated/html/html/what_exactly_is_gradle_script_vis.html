Gradle Script Visualizer

What exactly is the Gradle Script Visualizer?

Ever take a look at an Gradle build script and, although perfectly readable, not really "see" all the dependencies between your tasks?

And any imported Gradle scripts - where do they fit in? This program was designed to show that, by importing your Ant scripts and creating graphic file representations of them. Say what?

Features

  • Parses Gradle build files for the following task usages:
    • Task declarations
    • Task dependsOn
  • Apply from: for imported files from the file system
  • Apply from: for imported files from the a URL
  • Optional real-time updating of graphs as the build files are modified (you do have to keep the program running for this(
  • Optionally group results by build file subgraphs, or all items together in one graph
  • Output formats: PNG (default for PC), PDF (default for OS X, not available otherwise)

    • Gradle Script Visualizer is a program I've written to make visualizing how your Gradle tasks and scripts are related to one another.

    This was originally written as an exercise in TDD (Test Driven Development), I wanted to see how it affected development time compared to the very similar AntScriptVisualizer program on this site, which was written more traditionally. Guess what? TDD was way, way faster.
